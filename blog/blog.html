SCOPE of `var`, `let`, and `const`
var, let, and const are keywords used for declaring variables in JavaScript, and their scope determines where the variable can be accessed and manipulated within our code.

var: Variables declared with var are function-scoped. They are accessible within the function where they are defined, but not outside of it.

let: let are block-scoped. They are accessible within the block (e.g., a loop or conditional statement) where they are defined.

const: Variables declared with const are also block-scoped, just like let.const variables cannot be reassigned once they are assigned a value.
Use cases of `null` and `undefined`


Undefined:

Default Value: Variables that are declared but not initialized are automatically assigned the value undefined.
Function Return Value: If a function does not explicitly return a value, it implicitly returns undefined.
Property Access: If you attempt to access an object property that doesn't exist, it returns undefined.
Function Parameters: If a function is called with fewer arguments than declared parameters, the missing parameters are undefined.
Variable Scope: Variables declared without a value initialization are initially undefined.

Null:

Explicit Absence of Value: null is a value that represents the explicit absence of any object value or no value.
Typically Set by Developers: Unlike undefined, null is typically assigned by a developer to indicate that a variable or object property intentionally has no value or is empty.
Useful for Clearing References: It can be used to clear a reference to an object, indicating that it should no longer be used.

REST API
REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing networked applications. REST is not a protocol but a set of constraints and principles that provide a lightweight and scalable approach to designing web services. RESTful APIs use standard HTTP methods to perform CRUD (Create, Read, Update, Delete) operations on resources.
The commonly used HTTP methods include:

GET: Retrieve data (Read).
POST: Create new data (Create).
PUT: Update existing data (Update).
DELETE: Remove data (Delete).
